# Лабораторная работа №3. Настройка среды разработки

## 1. Используемые программы.    
### На постоянной основе я пользуюсь следующими программами:
- __редактирование текстовых файлов:__ Visual Studio Code;
- __программирования на основном языке программирования:__ Visual Studio Code;
- __для выполнения команд в консоли:__ WSL (Windows Subsystem for Linux), командная строка Windows Terminal;
- __для навигации по файловой системе и манипуляций с файлами:__ проводник Windows, командная строка, Visual Studio Code;
- __для работы с гитом:__ Git Bash, Visual Studio Code.
### Альтернативы:
- __Visual Studio Code:__ Visual Studio;
- __Visual Studio Code (для текста):__ Vim, Neovim, Sublime Text;
- __Windows Terminal:__ Windows PowerShell;
- __Git Bash:__ GitHub Desktop.

На протяжении 2 недель я редактировала текстовые файлы в Sublime Text.  
Эта программа имеет более простой и интуитивно понятный интерфейс, нежели VS Code. Однако редактируя текстовые файлы для последующей их загрузки на Git, нерационально использовать Sublime Text, так как в базовой установке Sublime Text нет встроенной интеграции с Git. Такую возможность предоставляет VS Code.
## 2. Look and Feel
### 2.1 Цвета интерфейса и шрифты
VS Code: необходимо зайти в _File -> Preferences -> Settings ->_  
- _Workbench -> Appearance_: 
    - с помощью настройки _Color Theme_ можно выбрать цвета интерфейса из предложенных шаблонов (по умолчанию стоит Dark Modern). Перейти к этой настройке можно, нажав сочетание клавиш `Ctrl+K` `Ctrl+T`;
    - с помощью настройки _Color Customizations_ можно внести изменения в файл settings.json (содержит пользовательские настройки VS Code) и настроить цветовую гамму интерфейса под себя. Также во вкладке _Extensions_ (Ctrl+Shift+X) на сайдбаре можно найти расширения с шаблонами цветов интерфейса и установить их.
- _Text Editor -> Font_: можно изменить шрифт и его размер.  

В интерфейсе своей IDE я изменила размер шрифта на 15 и в качестве "цветовой темы" поставила Dark+.
### 2.2 Расположение элементов интерфейса
В VS Code переместить панели можно, перетащив их в нужное место.  
Также настройки расположения панелей можно найти в верхней панели _View -> Appearance_ и отрегулировать их расположение относительно друг друга, а также увеличить масштаб.  
Справа (по умолчанию) в верхней панели есть значок настройки _Customize Layout_, в которой можно скрыть некоторые панели, а также выбрать расположение элементов интерфейса (каждой из панелей).  

По умолчанию стоят следующие настройки:
- Visibility (видимость): all;
- Primary Side Bar Position (расположение сайдбара): Left;
- Panel Alignment (выравнивание панелей): Center;
- Quick Input Position (расположение "быстрого ввода"): Top.  

Для настройки расположения панелей в интерфейсе своей IDE я изменила Panel Alignment на Justify, чтобы убрать ненужное пространство на сайдбаре слева и увеличить окно терминала.
### 2.3 Приглашение ко вводу
Когда в терминале VS Code открыт WSL можно настроить приглашение ко вводу следующим образом: 
1. Открыть файл Bash в редакторе nano `nano ~/.bashrc`. В нём настраиваются внешний вид приглашения и переменные окружения
2. В конец текстового файла добавить `PS1='\[\e[32m\]\u@\h \[\e[34m\]\W \[\e[33m\]$(__git_ps1 "(%s)")\[\e[0m\]\$ '`:
    - PS1 - переменная, которая отвечает за приглашение командной строки
    - `\e` - escape-последовательность
    - `32` - код зеленого цвета, `34` - код синего цвета, `m` - применить цвет
    - `\u` - имя пользователя
    - `\h` - имя хоста
    - `\w` (полный путь) или `\W` (текущая директория)
    - `$(__git_ps1 "(%s)")` - отображение ветки Git (проверяет, находится ли пользователь в git-репозитории)
    - `\[\e[0m\]` - сброс цвета
    - `\$` - символ приглашения
3. Сохранить изменения:
    - `Ctrl+O` - сохранить
    - `Enter` - подтвердить имя файла .bashrc
    - `Ctrl+X` - выйти из редактора
4. Применить изменения и ввести в терминал `source ~/.bashrc` (выполнение файла .bashrc заново)
## 3. Эргономика работы с кодом
Горячие клавиши в VS Code можно посмотреть, перейдя по пути _File -> Preferences -> Keyboard Shortcuts_ или нажав подряд сочетания `Ctrl+K`+`Ctrl+S`.  

#### Некоторые полезные горячие клавиши:
- `Ctrl+T` - поиск по символу;
- `Alt+Up/Down` - переместить строку выше или ниже;
- `Shift+Del` - удалить всю строку целиком;
- `Ctrl+Alt+RightArrow` - открыть текущую вкладку в новом окне;
- `Ctrl+P`- поиск файла по имени.

| Функция | Горячие клавиши|
|:--------|:-----------|
|поиск по содержанию файлов | __Ctrl+Shift+F__ |
|перейти к объявлению функции или типа | __F12__ или __Ctrl+Rclick__|
|перейти к реализации функции или типа | __Ctrl+F12__ |
|найти все места, где используется переменная, функция или тип | __Shift+F12__ |
|переименовать все вхождения переменной, функции или типа | __F2__ |
|быстро перемещаться между несколькими наиболее интересными местами кода | __Alt+RightArrow__ или __Alt+LeftArrow__ |
|комментировать и раскомментировать большие части кода | __Ctrl+/__ |
## 4. Кастомизация процессов
Для того, чтобы настроить собственные сочетания клавиш, нужно открыть список горячих клавиш _File -> Preferences -> Keyboard Shortcuts_ или нажав подряд сочетания `Ctrl+K`+`Ctrl+S`. Необходимо нажать на значок изменения в левом столбце таблицы рядом с названием команды и ввести нужное сочетание.
#### Сочетания клавиш после настройки для:
- `Ctrl+H` - сборки программы в нужной среде (кастомизация);
- `F5` - запуска программы в режиме отладки или релиза;
- `Ctrl+; A` - запуска автоматических тестов проекта.
## 5. Интеграция с гитом
В VS Code есть встроенная интеграция Git.
Действия с репозиторием можно делать как напрямую из консоли, так и из панели _Source Control_, её можно найти в левой части экрана или открыть, нажав `Ctrl+Shift+G`.
- Изменения можно увидеть в разделе _Source Control -> Changes_. 
- Чтобы добавить файлы, нужно нажать на "+" рядом с именем файла в разделе _Changes_.
- Для создания коммита нужно сначала ввести его название в строке _Message_ и нажать кнопку Commit.
- Чтобы добавить удаленный репозиторий, нужно в терминале VS Code ввести команду `git remote add origin` и указать ссылку на репозиторий.
- Для отправки/получения изменений в том же разделе необходимо выбрать параметры _Push/Pull_. Также можно сразу отправить изменения в удаленный репозиторий, выбрать в параметрах коммита _Commit & Push_.
- При возникновении конфликта в окне VS Code появится конфликтный файл. В нём будут выделены различия, а также будет показываться несколько параметров:
    - Accept Current Change (принять текущее изменение)
    - Accept Incoming Change (принять входящее изменение)
    - Accept Both Changes (принять все изменения)
    - Compare Changes (сравнить изменения)


    После выбора нужного параметра файл будет откредактирован IDE.
